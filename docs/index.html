$ clickfs▊

Mount ClickHouse as a filesystem.

Browse databases with ls. Stream tables with cat. No SQL boilerplate, no temp files.

cargo install clickfs
GitHub → Docs → crates.io →

Quickstart

  1. Install 
    cargo install clickfs
  2. Mount 
    clickfs mount http://localhost:8123 /mnt/ch &
  3. Use 
    ls /mnt/ch/db/default/
cat /mnt/ch/db/default/events/all.tsv | head
wc -l /mnt/ch/db/default/events/2024-01-15.tsv

How it works

Read the full architecture docs for the path-mapping rules, streaming bridge design, and SQL construction.

What it does

  • Each table appears as all.tsv plus one file per partition.
  • Reads stream from a single SELECT over HTTP — no buffering.
  • Strict-sequential reads per file descriptor; EIO on random seek.
  • Read-only: every mutating syscall returns EROFS.
  • Cancels via dropped HTTP connection plus server max_execution_time.
  • No client library, no daemon, no cache — one process, one mountpoint.
